# 0048 Benchmark-Governed First-Pass Review Adversarial Playbook

## Purpose

Record the adversarial checks that the `0048` spec and plan must continue to satisfy.

## Adversarial Checks

- Scope check: the package remains a repository-local first review pass rather than a claim of full discovery validation.
- Evidence check: every candidate record preserves fixture-bounded evidence labels, limitations, and the real-data rerun requirement.
- Rule check: review waves and dispositions are derived from explicit tracked fields rather than free-form narrative overrides, with the primary wave restricted to the declared transition-led rule.
- Anchor check: benchmark anchors are tied to tracked benchmark objects or validation packages that already exist in generated artifacts.
- Governance check: the first-pass package does not tune thresholds, objective weights, or hold-out rankings.
- Reporting check: the report distinguishes current review readiness from external scientific or publication readiness.

## Required Response

If any check fails, revise the spec, the plan, or the emitted package before treating the first pass as authoritative.

## Exit Condition

This playbook remains satisfied when the first-pass package can be rerun from tracked artifacts and still preserves those constraints explicitly.
