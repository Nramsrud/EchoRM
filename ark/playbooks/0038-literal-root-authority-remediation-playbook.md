# 0038 Literal Root Authority Remediation Playbook

## Purpose

Preserve the adversarial rubric used to prevent repository-local readiness from being overstated as literal full-scope closure.

## Current Use

- Treat this playbook as a claims-boundary checklist, not as a statement that the repository lacks the integrated root-closeout stack on `main`.
- When a document or report summarizes root-closeout status, it must preserve the evidence labels, limitations, and non-demonstrated capabilities from the generated packages.

## Non-Acceptable Evidence

- surrogate or proxy outputs promoted as literal backend completion
- fixture slices promoted as full corpus scope
- synthetic or literature-inspired support tasks promoted as full real-data validation
- optimization summaries promoted without benchmark and hold-out guards
- discovery rankings promoted without explicit evidence bundles and manual review
- release inventories promoted as external publication readiness without content review

## Review Rules

1. Benchmark, discovery, and release claims must cite the package that demonstrates them.
2. Discovery language must preserve hold-out governance and manual-review requirements.
3. Release language may describe repository-local bundles, not external publication status, unless the generated bundle is explicitly reviewed and accepted.
4. Root-closeout language may describe repository-local readiness over tracked artifacts, not external peer-review completion.

## Exit Condition

This playbook remains satisfied when repository documentation continues to preserve those claims boundaries.
