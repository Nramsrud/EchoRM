# 0001 Foundation Baseline

## Summary

Establish the minimal public repository baseline: Python packaging, verification, publication standards, and a numbered work structure derived from the authority documents.

## Scope

- Establish Python project metadata and quality tooling.
- Define the boundary between committed public artifacts and local private artifacts.
- Preserve the authority documents and create the initial numbered work structure.
- Keep large generated artifacts outside normal versioned content.

## Non-Goals

- Implement ingestion, reverberation-mapping, or sonification features.
- Stand up the full workflow and experiment stack.
- Reorganize the root scientific planning documents.
- Add raw datasets or generated artifacts.

## Global Constraints

- All committed outputs must remain formal, concise, precise, and limited to essential content.
- Committed text may reference only committed files or widely understood public concepts.

## Acceptance Criteria

- Python package metadata, tests, linting, type-checking, and CI are defined and pass.
- Public documentation states the publication boundary and writing standard.
- The numbered work structure exists and can host decomposed specifications and plans.
- Local private artifacts remain outside normal versioned content.
