# Lessons Learned

Repository-specific notes to check before drafting or implementing future EchoRM work.

## 2026-03-18 - Foundation baseline

- Maintain a strict separation between public committed documentation and local private tooling.
- Apply the local documentation standard to every committed artifact.
- Prefer narrow work packages with explicit dependencies over broad phase-wide bundles.
- Keep large raw data, rendered audio, and other generated artifacts outside normal versioned content.

## 2026-03-19 - Benchmark scope discipline

- Do not describe operational readiness as scientific benchmark completion.
- Label evidence levels explicitly when combining fixture-backed real-data checks and synthetic validation.
- Require benchmark dossiers to state demonstrated capability, limitations, and next scientific gaps directly.
